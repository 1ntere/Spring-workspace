스프링 부트 프로젝트 실행방법(서버 배포 방법)

	src/main/java 하위 패키지에
	ㅁㅁㅁApplication.java 파일이 프로젝트 생성시 자동으로 생성됨
	(ㅁㅁㅁ 은 프로젝트 파일명)
	
	해당 클래스는 main 메서드 존재
	-> 실행 -> Tomcat 서버가 안에 존재하기 때문에 알아서 배포 시작
	(브라우저가 알아서 실행되지 않기 때문에 각자 주소를 알아서 입력)
	http://localhast:서버포트/주소값
	
	우리는 8081 서버포트를 줬기 때문에
	http://localhost:8081
	에러페이지 현재는 발생!

프로젝트 구조

  • src/main/java : 자바 소스코드를 모아두는 폴더
  • src/main/resources : 자바에서 사용할 자원, 설정 파일을 모아두는 폴더
    • templates : java 템플릿(응답 화면)을 모아두는 폴더 주로 html 존재
    • static : templates에 적용될 정적 자원을 모아두는 폴더 주로 javascript, css, 이미지 등
    • application.properties : ★SpringBoot 프로젝트 관련 설정을 정의하는 파일 (심하게 중요함)!!!
    • maven : Apche에서 개발한 프로젝트 관리 및 이해도구 주로 pom.xml 파일로 존재 추가하고 싶은 기능을 작성
    • gradle : 그루비를 이용한 빌드 자동화 시스템 build.gradle 파일로 존재 추가하고 싶은 기능을 작성 그루비 : 자바에 파이썬이나 루비 등의 특징을 더한 언어 코틀린을 이용한 빌드 자동화 시스템 코틀린 : 웹 앱을 만드는 언어
